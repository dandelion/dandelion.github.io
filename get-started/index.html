Dandelion Core

First it might be a good idea to familiarize yourself with the key concepts of Dandelion.

For example:

  • What is an asset bundle
  • How to use an asset locator

The next step would be to give a try to the Dandelion samples. In general, all sample applications have been splitted into multiple tiny apps in order to focus on a particular feature. Feel free to browse the sample on GitHub to have all needed information.

You can also give a try to the Dandelion version of the Spring PetClinic application (TODO). In this version, all assets are based on Dandelion, so if you are already familiar with Spring and this sample application, this is a really good starting point.

Dandelion-Datatables

Dandelion-Datatables is the first and most advanced component of the framework focused on HTML tables.

Lots of samples are available, each of them focused on a particular feature or a set of related features. Feel free to browse/clone/fork them!

Note that you can also take a look at the official Spring PetClinic sample application that ships Dandelion-Datatables in its technology stack.
