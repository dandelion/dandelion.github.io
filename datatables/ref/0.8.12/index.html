Dandelion-Datatables 0.8.12 05-03-2013

Change log

Core

  • `dandelion.dev.mode` enabled by default

    Now you don't need to set the dandelion.dev.mode system property to `true` to disable caching. Caching is disabled by default, which means the DataTables configuration is computed each time you access a page.

  • Performance improvement

    The `DisplayType` object, used in export, is now instantiated much less time than before.

JSP implementation

  • New {% refjsp column visible %} column attribute, allowing you to hide column. Note that a hidden column is automatically unsearchable

Thymeleaf implementation

  • {% reftl table dt:export %}, {% reftl table dt:exportLinks%} and {% reftl table dt:filterType %} attributes don't expect a Thymeleaf expression any longer. Actually you can use both header-coded values and Thymeleaf Standard expressions
  • New {% reftl table dt:visible %} column attribute, allowing you to hide column. Note that a hidden column is automatically unsearchable
