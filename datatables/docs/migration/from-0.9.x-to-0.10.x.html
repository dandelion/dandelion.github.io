Migrating from 0.9.x to 0.10.x

As with the previous release, much has changed. You will find below as exhaustive a list as possible of all changes.

Configuration properties

Almost all the internal configuration has been refactored. As a consequence, if you were previously using a custom datatables.properties file, please update it following the configuration reference.

JSP taglib

Some attributes have been renamed for better consistency, while others have been removed or added.

You will find below all renamed and removed attributes. Please see the change log of the v0.10.0 for the added attributes.

Renamed attributes:

  • filter => filterable
  • sort => sortable
  • paginate => pageable
  • sortInit => sortInitDirection

Removed attributes:

  • scroller, colreorder and fixedheader: all supported plugins can now be enabled thanks to the ext table attribute (lien)
  • cdn: the asset provisionning is now handled by Dandelion.

Thymeleaf dialect

As for the JSP taglib, there was some changes in the Thymeleaf dialect.

Renamed attributes:

  • dt:filter => dt:filterable
  • dt:sort => dt:sortable
  • dt:paginate => dt:pageable
  • dt:sortInit => dt:sortInitDirection

Removed attributes:

  • dt:scroller, dt:colReorder and dt:fixedHeader: all supported plugins can now be enabled thanks to the dt:ext table attribute (lien)
  • cdn: the asset provisionning is now handled by Dandelion.

Configuration properties

All Dandelion modules now uses a unified system for loading configuration. As a consequence, each file must be in its place.

For Dandelion, the dandelion.properties must locate under the /dandelion directory.

For Dandelion-Datatables, all *.properties files must now locate under the /dandelion/datatables directory. Apart from that, the configuration loading strategy (lien) remains applyable.

Moreover, some properties have changed. Please see the reference for an exhaustive list of all available configuration properties. (lien)

Using plugins and export with Thymeleaf

  • Most of attributesAll export attributes has been moved from the <tbody> tag to the <thead> tag.
  • The value of some attributes don't require to be surrounded by single quotes anymore. See the explanations here.

Servlet2 extra

The required Servlet API version has been downgraded to 2.x in the v0.10.0. We directly integrated the Servlet 2.x-compatible classes into Dandelion-Datatables' core. As a consequence, the Servlet2 extra has been removed.

Please take a look at the updated installation guides.
