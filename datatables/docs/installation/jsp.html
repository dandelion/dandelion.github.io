Installation guide with JSPs

Step 1: Add the needed dependency

Manually

Download the needed JARs and make sure that they are all made available via the application classpath.

Using Maven or another build tool

{% assign artifact = 'datatables-jsp' %} {% include build-tools.html %}

Step 2: Configure the Dandelion filter

Add the following configuration to your web.xml file: 

<!-- Dandelion filter definition and mapping -->
<filter>
   <filter-name>dandelionFilter</filter-name>
   <filter-class>com.github.dandelion.core.web.DandelionFilter</filter-class>
</filter>
<filter-mapping>
   <filter-name>dandelionFilter</filter-name>
   <url-pattern>/*</url-pattern>
</filter-mapping>

Step 3: Configure the Dandelion servlet

Add the following configuration to your web.xml file: 

<!-- Dandelion servlet definition and mapping -->
<servlet>
   <servlet-name>dandelionServlet</servlet-name>
   <servlet-class>com.github.dandelion.core.web.DandelionServlet</servlet-class>
</servlet>
<servlet-mapping>
   <servlet-name>dandelionServlet</servlet-name>
   <url-pattern>/dandelion-assets/*</url-pattern>
</servlet-mapping>

Step 4: Declare the taglib definition

Everywhere you want to display a table using Dandelion-DataTables, you need to add the taglib definition in your JSP: 

<%@ taglib prefix="datatables" uri="http://github.com/dandelion/datatables" %>

You can now begin using the taglib!

Cient-side dependencies

Dandelion-Datatables needs some client-side libraries such as DataTables and jQuery to work.

Starting from the v0.10.0, Dandelion-Datatables is based on the asset bundles managed by Dandelion Core. Therefore, it ships a couple of vendor bundles, i.e. bundle containing vendor libraries, all configured to use a CDN.

This simply means that once the installation steps performed, it just works.

Of course, you may want to use your own assets instead. In this situation, you have several options:

  • You may need to make a more fine-tuned configuration of the assets. Read more here.
  • Or you may want to use Dandelion-Datatables as a standalone library, i.e. without the asset management brought by Dandelion. Read more here.
