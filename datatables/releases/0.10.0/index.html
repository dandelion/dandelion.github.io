Dandelion-Datatables 0.10.0 change log11-24-2013

Core

  • Finally connected with Dandelion Core!
    • All extensions have been refactored in order to work with the AssetStack (lien)
    • Lots of Dandelion scopes have been added and are available in the JAR:
      • jQuery
      • jQuery UI
      • Bootstrap v2
      • Bootstrap v3
      • DataTables
      • See the full list here (lien)
  • Refactored the table and column configurations processing for greater clarity and better performance
  • Cleaned Javascript generation: unneccessary parameters are non longer generated
  • Added support for pluging a custom Javascript generator (lien)
  • Refactored the Javascript generation to be able to compute the code before rendering it
  • Improved exception handling
  • Added support for hot reloading of the whole configuration, i.e.:
    • datatables.properties: the main configuration file
    • all datatables_XX.properties: bundles containing internationalized messages
  • Added new themes for Bootstrap Responsive and Bootstrap 3
  • Fixed display issue in the listbox pagination style
  • Added URL processing (back)
  • Added support for totally custom export (custom format, custom class, ...) (lien)
  • New pagination styles based on Bootstrap 2: bootstrap_four_button and bootstrap_full_numbers
  • Fixed usage of BEFORESTARTDOCUMENTREADY when using extra Javascript files (lien)
  • Better test coverage

JSP implementation

  • New id (lien) attribute for the <datatables:column> tag, allowing to assign an ID to the header cell
  • New filterMinLength (lien) attribute for the <datatables:column> tag, allowing to trigger the filtering after a minimum number of characters has been reached in a filtering field.
  • New filterDelay (lien) attribute for the <datatables:column> tag, allowing to set a delay (in ms) before the AJAX call is performed to obtain data from server
  • New filterSelector (lien) attribute for the <datatables:column> tag, allowing
  • New filterClearSelector (lien) attribute for the <datatables:column> tag, allowing
  • New name (lien) attribute for the <datatables:column> tag (lien), allowing to apply a name to a column. Used with server-side processing.
  • New escapeXml (lien) attribute for the <datatables:column> tag, allowing to disable the XML characters escaping
  • New <datatables:extraHtml> (lien) tag, allowing to create DataTables feature plug-ins that can be used to decorate the table. See more details here. (lien)
  • New orientation (lien) for the <datatables:export> tag (lien), allowing to change the default PDF orientation
  • New fileExtension (lien) for the <datatables:export> tag (lien), allowing to set the extension of the exported file
  • New mimeType (lien) for the <datatables:export> tag (lien), allowing to set the mime Type of the response when exporting data
  • New escapeXml (lien) attribute for the <datatables:export> tag, allowing to disable the XML characters escaping
  • New reloadSelector (lien) attribute for the <datatables:table> tag, allowing to bind a "click" event to element that will trigger the table reloading
  • New reloadFunction (lien) attribute for the <datatables:table> tag, allowing to customize the table reloading
  • Renamed the sortInit table attribute to sortInitDirection
  • New sortInitOrder (lien) attribute for the <datatables:table> tag, allowing to set the order in which columns should be initialized
  • New escapeXml (lien) attribute for the <datatables:table> tag, allowing to disable the XML characters escaping
  • Renamed the rowIdSufix table attribute to rowIdSuffix
  • Renamed the filter table attribute to filterable
  • Renamed the sort table attribute to sortable
  • Renamed the paginate table attribute to pageable
  • Removed the unused footer table attribute
  • Removed the <datatables:extraConf> tag
  • Generally speaking, <rtexprvalue>true</rtexprvalue> has been added in all TLDs where it was missing

  • Added support for Thymeleaf 2.1.x
