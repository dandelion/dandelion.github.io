Available export formats

Preamble

The following formats are available: CSV, XML, PDF, XLS and XLSX.

Note that depending on the export formats you allow in a table, you may need some additional dependencies.

Text-based export formats: CSV, XML

For the text-based export formats, such as CSV or XML, no additional dependency is required.

You have two options to generate export files:

  • Either you use the default export classes. Indeed Dandelion-Datatables includes some export classes that are configured by default: {% javadoc dt datatables-core CsvExport %} and {% javadoc dt datatables-core XmlExport %}.
  • Or you configure other export classes. See this tutorial to know how to proceed.

Binary-based export formats: XLS, XLSX, PDF

Binary export formats, such as XLS or PDF, require most of the time some additional dependencies.

Note that Dandelion-Datatables also includes some extras that you can use in these cases:

  • datatables-export-itext: used for PDF exports
  • datatables-export-poi: used for XLS exports
  • datatables-export-poi-ooxml: used for XLSX exports (Excel 2007 or later)

Each of those dependencies contains:

  • an export class that will be used by default by Dandelion-Datatables if the corresponding export format is allowed in the table.
  • the corresponding third-party dependency (e.g. the iText library for the datatatables-export-itext extra)
