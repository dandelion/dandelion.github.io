Overview of the extension mechanism

Since the earliest version of Dandelion-Datatables, we've been trying to develop the framework in a modular manner. That's why if you dig into the source code, you'll see lots of already existing extensions as the {% javadoc dt datatables-core AjaxFeature %}, the {% javadoc dt datatables-core ServerSideFeature %}, or the {% javadoc dt datatables-core Bootstrap2Theme %}, ...

Starting from the v0.9.0, you can now write your own extensions, providing a powerful mechanism to customize your tables.

What is an extension?

Any extension will generally fall into one of three categories:

  • Theme: extension used to theme a table (e.g. {% javadoc dt datatables-core Bootstrap2Theme %})
  • Plugin: extension used to encapsulate a DataTables' native plugin (e.g. {% javadoc dt datatables-core ScrollerPlugin %})
  • Feature: all other extensions (e.g. {% javadoc dt datatables-core ServerSideFeature %})

But generally speaking, we'll call it simply extension.

What you can do with extensions

Access the DataTables' parameters

One of the key features of Dandelion-Datatables is that it generates all the needed DataTables parameters for you, depending on how you used the JSP taglib or the Thymeleaf dialect.

By using the extension mechanism, you can easily modify these parameters:

  • by adding or replacing any of them
  • by adding a DataTable callback
  • by chaining another Javascript call (useful for extra, such as (lien))

Access the Dandelion scopes

Often the extension needs some extra Javascript code, e.g. a library.

The concept of scope (lien) introduced in the v0.10.0 is also accessible from any extension. Thus, you can benefit from the powerful asset system of Dandelion and play with scopes inside your extension.

Access the whole generated Javascript file

Any Javascript code can be inserted in the generated Javascript file. Useful if you need for example to set a variable before the DataTables initialization.

Access the HTML table

The HTML table can be easily accessed, as well as the columns and all other nested tags.


To summarize, you have here a powerful mechanism in situations where all ready-to-use extensions don't fit your needs.
