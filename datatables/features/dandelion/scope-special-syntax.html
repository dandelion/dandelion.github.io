Scope special syntax

Since the v0.10.0, some Dandelion-Datatables features can directly impact the asset stack. Let's see how.

New scope syntax

Some features now accept the following syntax: 

attribute="scopeName#jsObject"

where:

  • scopeName is the name of the scope to add to the current request
  • jsObject is a JavaScript object: a function, an array, or any other Javascript object depending on the attribute

Of course you can also specify multiple scopes, comma-separated: 

attribute="scope1,scope2#jsObject"


Info!
Note that you could achieve the same goal by using the Dandelion JSP taglib or the Thymeleaf dialect. But this syntax is finally a convenient shortcut, preventing you from bringing another dependency and declaring another JSP taglib / Thymeleaf dialect.

Example

For example, you can use this new syntax in the {% refjsp dt table serverData %} / {% reftl dt table serverData %} table attribute, normally used to replace the default AJAX call by a totally customized one.

Assuming that the following function that is intended to make the AJAX call is stored in a Javascript file, which is part of a scope called my-scope, you could automatically load the right scope to the current request using: 

<datatables:table id="myTableId" url="/persons" serverSide="true" serverData="my-scope#myAjaxRequest">
   <datatables:column title="Id" property="id" />
   <datatables:column title="FirstName" property="firstName" />
   <datatables:column title="LastName" property="lastName" />
   <datatables:column title="City" property="address.town.name" />
   <datatables:column title="Mail" property="mail" />
</datatables:table>
