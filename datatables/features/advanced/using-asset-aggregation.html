Using asset aggregation
JSP Thymeleaf

Preamble

In some situations, you may end up having lots of assets to gather before initializing your table. The aggregation has been introduced to optimize performance during your page loading, aggregating all assets into one file, one for each asset type, allowing to reduce the number of HTTP requests.

Activating asset aggregation

You'll have to tell Dandelion-Datatables you want to aggregate assets. There are multiple ways for that:

  • Setting up the {% refjsp dt configuration main main.aggregator.enable %} property to true in your datatables.properties configuration file. Aggregation will be turned on in all your application.
  • Using the {% refjsp dt prop %} JSP tag locally, in a particular table.

Using JSP

Here is an example using the {% refjsp dt prop %} tag: 

<datatables:table id="myTableId" data="${persons}">
   <datatables:column title="Id" property="id" />
   <datatables:column title="FirstName" property="firstName" />
   <datatables:column title="LastName" property="lastName" />
   <datatables:column title="City" property="address.town.name" />
   <datatables:column title="Mail" property="mail" />
   <datatables:prop name="main.compressor.enable" value="true" />
</datatables:table>

Using Thymeleaf

For now, the only way to aggregate assets with Thymeleaf is to use the {% refjsp dt configuration main main.aggregator.enable %} property in your datatables.properties file.
