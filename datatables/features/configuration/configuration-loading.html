Configuration loading

Preamble

Dandelion-Datatables internally uses a configuration file called datatables-default.properties that contains configuration properties used by default.

These configuration properties can be overloaded in multiple ways. This section decribes how.

Loading strategy

You can overload the default configuration using a file named datatables.properties. Dandelion-Datatables will load it thanks to the ResourceBundle mechanism and using the following strategy:

  • First, check if the dandelion.datatables.configuration system property is defined.

    If it exists, Dandelion-Datatables attempts to convert the value to a URL and use it to load a resource bundle called datatables.properties

  • If no system property is defined, Dandelion-Datatables tries to load a resource bundle still called datatables.properties in classpath_root/dandelion/datatables/

  • If no datatables.properties file is found, only the default configuration will be used.

Note that this strategy can be overriden by pluging a custom configuration loader.

Example using classpath configuration

Let's say you use Maven as a build tool. When reading configuration from classpath, you should strictly follow the convention, i.e. place the properties files under the classpath_root/dandelion/datatables/ folder. 

   MyApp
   |__ src
      |__ main
         |__ java
         |__ resources
            |__ logback.xml
            |__ ...
            |__ dandelion
               |__ datatables 
                  |__ datatables.properties
                  |__ datatables_FR.properties
                  |__ datatables_ES.properties

Example using externalized configuration

Let's say you have the following files: 

   C:
   |__ Documents
      |__ MyApp
         |__ datatables.properties
         |__ datatables_FR.properties
         |__ datatables_ES.properties
You just need to add -Ddandelion.datatables.configuration="C:\Documents\MyApp\" to the starting script of the server.

Note that when reading from externalized file, you don't have to place the properties files under the dandelion folder.
