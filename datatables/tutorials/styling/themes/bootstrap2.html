Bootstrap 2 theme

Preamble

Dandelion-Datatables provides an easy way to apply the Bootstrap framework on your tables.

Activating the Bootstrap 2 theme

In order to activate the Bootstrap 2 theme, follow these steps:

  1. Provide the needed static assets

    Link to the Bootstrap 2 library (only the CSS is required)

    Of course, don't forget jQuery (JS) and DataTables (JS)

  2. Use the JSP taglib/Thymeleaf dialect

    Set the {% refjsp dt table theme %} (JSP) / {% reftl dt table dt:theme %} (Thymeleaf) table attributes to bootstrap2

  3. Use the Bootstrap classes

    At least, you need to add the table CSS class. But you can use any of the other CSS classes provided by Bootstrap: table-striped , table-bordered , ...

    Just use the {% refjsp dt table cssClass %} table attribute of the JSP taglib, or simply the class table attribute when using Thymeleaf.

Example

Using JSP

<link href="//netdna.bootstrapcdn.com/twitter-bootstrap/2.3.1/css/bootstrap-combined.min.css" rel="stylesheet">

<datatables:table id="myTableId" data="${persons}" theme="bootstrap2" cssClass="table table-striped">
   <datatables:column title="Id" property="id" />
   <datatables:column title="FirstName" property="firstName" />
   <datatables:column title="LastName" property="lastName" />
   <datatables:column title="Street" property="address.town.name" />
   <datatables:column title="Mail" property="mail" />
</datatables:table>

Using Thymeleaf

<link href="//netdna.bootstrapcdn.com/twitter-bootstrap/2.3.1/css/bootstrap-combined.min.css" rel="stylesheet">

<table id="myTableId" dt:table="true" dt:theme="bootstrap2" class="table table-striped">
   <thead>
      <tr>
         <th>Id</th>
         <th>Firstname</th>
         <th>Lastname</th>
         <th>Street</th>
         <th>Mail</th>
      </tr>
   </thead>
   <tbody>
      <tr th:each="person : ${persons}">
         <td th:text="${person.id}">1</td>
         <td th:text="${person.firstName}">John</td>
         <td th:text="${person.lastName}">Doe</td>
         <td th:text="${person.address.town.name}">Nobody knows !</td>
         <td th:text="${person.mail}">john@doe.com</td>
      </tr>
   </tbody>
</table>

Some notes...

Sorting icons

The DataTables' CSS is optional but if you don't use it, you will lose the sorting icons.

If you really need them, of course you can, by using the sorting CSS classes as follows: 

.sorting { background: url('/your-assets-directory/images/lib/datatables/sort_both.png') no-repeat center right; }
.sorting_asc { background: url('/your-assets-directory/images/lib/datatables/sort_asc.png') no-repeat center right; }
.sorting_desc { background: url('/your-assets-directory/images/lib/datatables/sort_desc.png') no-repeat center right; }
.sorting_asc_disabled { background: url('/your-assets-directory/images/lib/datatables/sort_asc_disabled.png') no-repeat center right; }
.sorting_desc_disabled { background: url('/your-assets-directory/images/lib/datatables/sort_desc_disabled.png') no-repeat center right; }

CDN feature

At the moment, you won't be able to use any custom style if you use the {% refjsp dt table cdn %} (JSP) / {% reftl dt table dt:cdn %} (Thymeleaf) table attributes.

Indeed, if you activate the CDN feature, Dandelion-Datatables will generate <script> and <link> tags and place them in a way yon't be able to override with your custom style.
