Internationalization
JSP Thymeleaf

Preamble

Since the v0.9.0, Dandelion-Datatables has now full support for internationalization of column titles and surrounding information.

Translating DataTables' information

All the i18n configuration takes place in the datatables.properties file. Lots of keys exist and allow you to configure the language information displayed by DataTables.

You can see the full reference here.

In order to support other languages, you can add any number of additional files names datatables_LANGUAGE.properties in the same place as the main datatables.properties file. For example, if your application needs to support both English and French, you could have the following files:

  • datatables.properties: the main configuration file, containing non-translatable properties 
    global.css.class=table table-striped table-bordered table-condensed
global.extra.theme=bootstrap2
  • datatables_EN.properties: the file containing all english translations 
    global.msg.processing=Processing...
global.msg.search=Search :
global.msg.info=Showing _START_ to _END_ of _TOTAL_ entries
  • datatables_FR.properties: the file containing all french translations 
    global.msg.processing=Traitement en cours...
global.msg.search=Rechercher :
global.msg.info=Affichage de l'élément _START_ à _END_ sur _TOTAL_ éléments

Note that you don't need to copy all the properties in all internationalized file because lots of them are also configuration properties which don't need to be translated at all.
