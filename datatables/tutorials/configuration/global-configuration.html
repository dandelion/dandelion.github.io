Using global configuration
JSP Thymeleaf

Preamble

Starting from the v0.9.0, almost all available attributes of the {% refjsp dt table %} and {% reftl dt table %} tags can be configured in the configuration file ( datatables.properties ) to be applied application-wide, i.e. on all tables in your application.

Since Dandelion-Datatables will defautly look for a file called datatables.properties at the root of the classpath, you just need to create it and write some key-value pairs inside.

All pairs you write in your file will override the default values contained in the default properties file which is embedded in the JAR. Nothing beats a nice drawing:

Feel free to browse the configuration reference to see what you can use globally.

Note that you can of course override any global configuration locally, using the corresponding attribute.

Example

Let's imagine you want to apply the Bootstrap2 theme everywhere in your application.

Before, you had to add the {% refjsp dt table theme %} attribute and add custom CSS classes in the {% refjsp dt table cssClass %} attribute, like this: 

<datatables:table id="myTableId" data="${persons}" cssClass="table table-striped table-bordered table-condensed" theme="bootstrap2">
   <datatables:column title="Id" property="id" />
   <datatables:column title="FirstName" property="firstName" />
   <datatables:column title="LastName" property="lastName" />
   <datatables:column title="Street" property="address.town.name" />
   <datatables:column title="Mail" property="mail" />
</datatables:table>

From now on you just need to fill the datatables.properties file with the following configuration: 

global.css.class=table table-striped table-bordered table-condensed
global.extra.theme=bootstrap2
And that's all! This way, the Bootstrap 2 theme will be applied on all tables in your application.
