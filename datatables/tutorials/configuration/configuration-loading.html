Configuration loading
JSP Thymeleaf

Preamble

By default, Dandelion-Datatables internally uses a configuration file (datatables-default.properties), containing configuration properties.

Those configuration properties can be overriden in multiple ways. This section decribes how.

Loading strategy

The configuration loading algorithm is defined as follows:

  1. First, check if the dandelion.datatables.configuration system property is defined.

    If it is, Dandelion-Datatables attempts to convert the value to a URL and tries to load a bundle called datatables.properties with it

  2. If no system property is defined, Dandelion-Datatables tries to load a bundle still called datatables.properties from the root of the classpath

Example using externalized configuration

Let's say you have the following files: 

   C:
   |__ Documents
      |__ MyApp
         |__ datatables.properties
         |__ datatables_FR.properties
         |__ datatables_ES.properties
You just need to add -Ddandelion.datatables.configuration="C:\Documents\MyApp\" to the command that starts the server.

Example using classpath configuration

Let's say you're developing a project and building it with Maven. You should have the following structure: 

   MyApp
   |__ src
      |__ main
         |__ java
         |__ resources
            |__ datatables.properties
            |__ datatables_FR.properties
            |__ datatables_ES.properties
