Configuration loading
JSP Thymeleaf

Preamble

Dandelion-Datatables internally uses a configuration file (datatables-default.properties), containing configuration properties that are used by default.

Those configuration properties can be overriden in multiple ways. This section decribes how.

Loading strategy

You can override the default configuration using a file named datatables.properties. Dandelion-Datatables will load it thanks to the ResourceBundle mechanism and using the following strategy:

  • First, check if the dandelion.datatables.configuration system property is defined.

    If it exists, Dandelion-Datatables attempts to convert the value to a URL and use it to load a resource bundle called datatables.properties

  • If no system property is defined, Dandelion-Datatables tries to load a resource bundle still called datatables.properties from the root of the classpath

  • If no datatables.properties is found, only the default configuration will be used.

Note that this strategy can be overriden by pluging a custom configuration loader.

Example using externalized configuration

Let's say you have the following files: 

   C:
   |__ Documents
      |__ MyApp
         |__ datatables.properties
         |__ datatables_FR.properties
         |__ datatables_ES.properties
You just need to add -Ddandelion.datatables.configuration="C:\Documents\MyApp\" to the command that starts the server.

Example using classpath configuration

Let's say you use Maven to build your project. You should have the following structure: 

   MyApp
   |__ src
      |__ main
         |__ java
         |__ resources
            |__ datatables.properties
            |__ datatables_FR.properties
            |__ datatables_ES.properties
