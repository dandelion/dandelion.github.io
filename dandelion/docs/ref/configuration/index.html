Configuration properties reference

Listed below are all the configuration properties that you can use in the configuration files.

Name Description Value(s) Default dev Default prod
minimification Enable the asset minification true | false false true
override.servlet3 Flag used to force Dandelion to locate assets with Servlet 2.x or Servlet 3.x compatible locators true | false
asset.locations.resolution.strategy Resolution strategy of asset locations if assets are defined with multiple locations Comma-separated list of location keys webapp,webjar,jar,cdn webapp,webjar,jar,cdn
asset.processors List of processors applied on each asset if minification is enabled. Comma-separated list of processor keys cssurlrewriting,jsmin,cssmin cssurlrewriting,jsmin,cssmin
asset.processors.encoding Encoding used by processors to write/read in/from streams UTF-8 UTF-8
asset.js.excludes Comma-separated list of JavaScript assets to always exclude from the graph
asset.css.excludes Comma-separated list of CSS assets to always exclude from the graph
bundle.includes Comma-separated list of bundles to include in all requests
bundle.excludes Comma-separated list of bundles to exclude from all requests
cache.name Name of the cache system to use. If nothing is set, Dandelion will use the default implementation default | ehcache | custom
cache.asset.max.size Max number of assets stored in the configured cache system 50 50
cache.request.max.size Max number of request/assets associations stored in the configured cache system 50 50
cache.manager.name Name of the cache manager to use
cache.configuration.location Location of the cache configuration file
