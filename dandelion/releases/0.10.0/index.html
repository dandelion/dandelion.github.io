Dandelion 0.10.0 change log??-??-2014

Core

  • Lots of internal features are now based on the Service Provider Interface API: processors, wrappers, locations and caching in order to provide plenty of extension points
  • The asset rendering is now based on a servlet filter (AssetFilter (plugin jekyll)), which optimally injects <script> and <link> tags into the page. Read more here. (lien)
  • New asset processors available for compression (based on YUICompressor) and aggregation. Read more here.
  • Added a new CdnLocationWrapper used to rewrite URLs in CSS files. Automatically activated when minification or aggregation is enabled.
  • The WebjarsLocationWrapper now support URL rewriting in CSS files when necessary.
  • New asset stack-related utilities:
    • Orphan bundles (lien) are now handled (dans quel sens ?)
    • Possibility to exclude assets by type
    • The asset position can be forced using a new JSON parameter (dom)
    • Possibility to activate bundles permanently (e.g. handy for global CSS files)
  • The old DatatablesController (used to serve generated assets of the Dandelion-Datatables component) has been repatriated to Dandelion core, renamed to AssetServlet (lien) and is now used to serve all kinds of generated assets (minified, aggregated, delegated, templated)
  • Dropped Servlet 3.x annotations in favour of plain old XML.
  • The URL of generated asset (minified, aggregated, delegated, templated) is now built with a SHA1 key instead of the old parameters previously used in Dandelion-Datatables.
  • Lots of new utilities for dealing with:
    • resource scanning
    • files
    • ...
  • All *AssetLoader are now mutually exclusives in order to be able to disable one or more loaders (useful for Dandelion integration in legacy apps)
  • Bundle and asset names are not case sensitive any longer
  • Fixed issue with protocol-relative URLs used as asset locations (aka //cdn)
  • DevMode refactored to avoid re-initializing the asset stack at each request thus drastically improving performance

Spring3 extra

  • The Spring3 extra now embeds a new AssetController (lien) that can be used as an alternative to the AssetServlet (lien). The controller has been bundled in a configuration file that can be easily imported in the main Spring configuration.

Ehcache extra

  • A new dandelion-ehcache module has been added, allowing to plug an Ehcache-based cache in the Dandelion cache system (lien).
