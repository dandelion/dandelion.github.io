Introducing asset location wrappers

Dandelion can build an asset stack with assets coming from multiple locations. This feature is based on asset location wrappers which will be introduced in this section.

What is a location wrapper?

An asset location wrapper (or location wrapper to simplify the language) is a component that will wraps the passed in location and process it in order to make it understandable by Dandelion.

The type of processing differs according to the configured location wrapper(s).

Consider the following bundle. 

{
   "bundle" : "jquery",
   "parent": "none",
   "assets": [{
      "name": "jquery-js",
      "version": "1.11.0",
      "type": "js",
      "locations": {
         "cdn": "//cdnjs.cloudflare.com/ajax/libs/jquery/1.11.0/jquery.js",
      }
   }]
}

This bundle defines only one asset called jquery-js with a single location, pointing to a CDN.

But Dandelion also allows to declare multiple locations. For example: 

{
   "bundle" : "jquery",
   "parent": "none",
   "assets": [{
      "name": "jquery-js",
      "version": "1.11.0",
      "type": "js",
      "locations": {
         "webapp" : "/assets/js/jquery.js",
         "cdn": "//cdnjs.cloudflare.com/ajax/libs/jquery/1.11.0/jquery.js"
      }
   }]
}

Both webapp and cdn are used in the declaration of the jquery-js asset.

Locations resolution strategy

By default, the following resolution strategy is used when multiples locations are declared:

  • Dandelion checks first if a cdn location key is defined. If so, the corresponding location will be used by the asset stack.
  • Then, if the webapp location key is defined, the corresponding location will be used by the asset stack.

You can of course override this behaviour by using the assets.locations configuration property.

Just change the order of selection of location keys in your configuration file. 

assets.locations=webjars,cdn
