Introducing asset location wrappers

Dandelion provides a powerful mechanism to locate the assets before using them.

What is a location wrapper?

An asset location wrapper (or location wrapper to simplify the language) is a component that will transform the specified asset location in order to make it usable by Dandelion.

The kind of transformation differs according to the configured location wrapper.

Consider the default jquery scope bundled with the dandelion-core JAR. 

{
    "scope" : "jquery",
    "parent": "none",
    "assets": [
      {
          "name": "jquery-js",
          "version": "1.11.0",
          "type": "js",
          "locations": {
              "cdn": "//cdnjs.cloudflare.com/ajax/libs/jquery/1.11.0/jquery.js"
          }
      }
    ]
}

The interesting part here is the locations property.

Built-in location wrappers

Location key Name Active by default Description
webapp WebappLocationWrapper (lien) yes

Relates to assets located in the web applicatin directory, e.g. src/main/webapp in Maven projects.

Read more here.

classpath ClasspathLocationWrapper (lien) yes

Relates to assets located in the classpath, e.g. in a JAR.

Read more here.

cdn CdnLocationWrapper (lien) no

Relates to assets served by a Content Delivery Network.

Read more here.

webjars WebjarsLocationWrapper (lien) yes

Relates to assets served by a WebJar.

Read more here.

templated TemplatedLocationWrapper (lien) yes

Relates to assets located in the classpath and which contains one or more variables to be replaced.

Read more here.

delegated DelegatedLocationWrapper (lien) yes

Relates to assets whose location is delegated to another component, e.g. Dandelion-Datatables which generates assets.

Read more here.
