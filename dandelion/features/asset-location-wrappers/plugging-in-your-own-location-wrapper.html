Plugging-in your own location wrapper

You may wish to use a custom location wrapper. Let's see here how to proceed.

SPI for location wrapper

In order to plug-in your own location wrapper, just follow these steps:

  1. Create a class that extends AbstractAssetJsonLoader (lien) or directly implements AssetLoader (lien). 
    package com.foo.asset.loader.impl;

public class MyAssetLoader extends AbstractAssetJsonLoader {

   private static final Logger LOG = LoggerFactory.getLogger(MyAssetLoader.class);

	@Override
	public String getName() {
		return "myLoader";
	}

	@Override
	public boolean isRecursive() {
		return false;
	}

	@Override
	protected Logger getLogger() {
		return LOG;
	}

	@Override
	public String getPath() {
		return "myPath";
	}
}

  2. In the META-INF/services folder, create a text file (UTF-8 encoded) using the following convention: 

    yourJar
|__ src
   |__ main
      |__ resources
         |__ META-INF
            |__ services
               |__ com.github.dandelion.core.asset.loader.spi.AssetLoader

    Inside this file, just add the fully qualified name of your own implementation. For example: 

    com.foo.asset.loader.impl.MyAssetLoader

And that's all! Dandelion, thanks to the SPI mechanism, will automatically pick up your implementation and add it to the active asset loaders on the next restart.

Example with Dandelion-Datatables

Dandelion-Datatables is a nice example of how a custom asset loader can be plugged in the asset loader system.

Indeed, the Dandelion-Datatables component provides its own asset loader: DatatablesAssetJsonLoader (lien). This loaders only scans inside the dandelion/datatables folder.

You can find the SPI configuration in the sources.
