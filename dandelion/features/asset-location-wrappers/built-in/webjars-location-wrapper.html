WebJars location wrapper

Installation

This wrapper is bundled in the dandelion-webjars extra since it brings a new dependency to the webjars-locator project, internally used by this location wrapper to locate assets within WebJars.

{% assign artifact = 'dandelion-webjars' %} {% include build-tools.html %}

Usage

The WebJar location wrapper must be used whenever you need to load an asset stored inside a WebJar.

Location key webjars
Active by default yes
Corresponding value Name of the asset (path excluded but extension included)

Example

Assuming that the bootstrap-2.3.0.jar is present in your classpath, the bundle definition should look like: 

{
   "bundle" : "my-bundle",
   "assets": [{
      "name": "bootstrap",
      "version": "2.3.0",
      "type": "css",
      "locations": {
         "webjars": "bootstrap.css"
      }
   }]
}
