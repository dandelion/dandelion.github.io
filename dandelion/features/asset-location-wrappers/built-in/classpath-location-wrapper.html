Classpath location wrapper

Usage

The classpath location wrapper must be used whenever you need to load an asset stored in the classpath.

Location key classpath
Active by default yes
Corresponding value Absolute name of the asset (no leading "/", as it internally uses ClassLoader.getResourceAsStream())

Example

Assuming the following project structure: 

your-project
|__ src
   |__ main
      |__ resources
         |__ js
            |__ application.js

The scope definition should look like: 

{
    "scope" : "my-scope",
    "parent": "none",
    "assets": [
      {
          "name": "my-application",
          "version": "1.0.0",
          "type": "js",
          "locations": {
              "webapp": "js/application.js"
          }
      }
    ]
}
