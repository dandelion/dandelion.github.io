CDN location wrapper

Usage

The CDN location wrapper must be used whenever you need to load an asset hosted by a Content Delivery Network.

Location key cdn
Active by default no
Corresponding value URL of the asset (protocal-relative URLs are supported)

The reason why this location wrapper is not active by default is that it is only useful whenever the loaded asset needs to be processed, e.g. whenever minification or aggegation is enabled.

In such cases, the CDN location wrapper will process all relative links inside CSS files before returning their content.

By default during the development phase, assets are not processed and the location configured in the JSON definition is left untouched in the HTML sources.

Example

{
   "bundle" : "jquery",
   "assets": [{
      "name": "jquery-js",
      "version": "1.11.0",
      "type": "js",
      "locations": {
         "cdn": "//cdnjs.cloudflare.com/ajax/libs/jquery/1.11.0/jquery.js"
      }
   }]
}
