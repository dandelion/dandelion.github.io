Built-in location wrappers

Listed below are the built-in location wrappers provided by Dandelion.

Location key Name Active by default Description
webapp WebappLocationWrapper yes

Relates to assets located in the web application directory, e.g. src/main/webapp in Maven projects.

Read more here.

classpath ClasspathLocationWrapper yes

Relates to assets located in the classpath, e.g. in a JAR.

Read more here.

cdn CdnLocationWrapper no

Relates to assets served by a Content Delivery Network.

Read more here.

webjars WebjarsLocationWrapper yes

Relates to assets served by a WebJar.

Read more here.

templated TemplatedLocationWrapper yes

Relates to assets located in the classpath and which contains one or more variables to be replaced.

Read more here.

delegated DelegatedLocationWrapper yes

Relates to assets whose location is delegated to another component, e.g. Dandelion-Datatables which generates assets.

Read more here.
