Webapp location wrapper

Usage

The webapp location wrapper must be used whenever you need to load an asset stored somewhere in the web application directory.

Location key webapp
Active by default yes
Corresponding value Relative path to the web application root directory

Example

Assuming the following project structure: 

your-project
|__ src
   |__ main
      |__ webapp
         |__ assets
            |__ js
               |__ application.js

The bundle definition should look like: 

{
   "bundle" : "my-bundle",
   "assets": [{
      "name": "my-application",
      "version": "1.0.0",
      "type": "js",
      "locations": {
         "webapp": "/assets/js/application.js"
      }
   }]
}
