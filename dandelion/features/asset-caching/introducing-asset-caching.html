Introducing asset caching

Dandelion provides some processors intended to be applied on assets. This section introduces major concepts around these processors.

Definition

An asset processor (or processor to simplify the language) is defined as follows:

  • A processor key, used as an identifier
  • A type of asset to process, such as js or css

All processors are currently used to minify assets thereby improving load times.

  In upcoming releases, more processors will be provided, allowing to process more CSS and JavaScript languages such as CoffeeScript.

Dandelion already provides some built-in processors which are configured by default when the production mode (TODO) is enabled. Note that when the development mode is enabled, the minification (TODO lien) configuration property is disabled, and thus no processor is applied.

Configuring asset processors

Asset processors are configured through the asset.processors (TODO) configuration property. You just need to define a comma-separated list of processor keys to enable the corresponding processors.

For example, the default value is cssurlrewriting,jsmin,cssmin . This means that by default, the processors behing these keys will be applied sequentially on all asset, depending on their compatible asset type(s).

Enabling asset processing

By default, no asset is processed, because the minification (TODO) configuration property is set to false (see the default configuration).

You can activate asset processing by turning the above configuration property to true . When set to true , Dandelion will use the asset processors configured thanks to the asset.processors (TODO) configuration property as seen in the previous section.

Asset caching and naming strategy

Once asset processing is enabled, there are two points to note:

  • Asset naming: the min keyword is appended to the asset name, just before the extension name.

    For example, an asset non-minified called app-1.0.0.css will be called app-1.0.0.min.css.

  • Asset caching: it is also worth noting that all processed assets will be first cached, thanks to the configured cache system, and then served by the DandelionServlet (TODO).
