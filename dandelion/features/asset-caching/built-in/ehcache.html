EhcacheAssetCache

If you already have Ehcache configured in your application, it can be handy to use it as a main cache system for Dandelion.

The dandelion-ehcache extra brings:

  • a new service provider for the {% javadoc ddl dandelion-core AssetCache %} SPI
  • a dependency to the Ehcache project

Cache name ehcache
Service provider {% javadoc ddl dandelion-extras/dandelion-ehcache EhcacheAssetCache %}

Installation

{% assign artifact = 'dandelion-ehcache' %} {% include build-tools.html %}

Once the artifact available in your classpath, you need to tell Dandelion that you want to use the service provider bundled inside.

For that, you just need to add the following configuration to your dandelion.properties file: 

cache.name=ehcache

Configuration

Dandelion comes with some configuration properties that allow to configure the Ehcache extra.

The first one is cache.configuration.location which allows to specify the location of the ehcache.xml file.

The second one is cache.manager.name which allows to specify the name of the cache manager to use to manipulate the cache.
