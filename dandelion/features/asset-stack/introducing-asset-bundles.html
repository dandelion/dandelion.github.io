Introducing assets and asset bundles

Asset and asset bundles are the main unit of work in Dandelion.

In web applications, assets are basically all the JavaScript and CSS files loaded in your web pages. Dandelion provides a wider representation of these assets, using bundles.

What is an asset bundle?

An asset bundle, or bundle to simplify the language, is a little more than just a set of assets, because it contains additional information.

All bundles have a JSON representation. For example, we could build a jquery bundle as follows: 

{
   "bundle" : "jquery",
   "parent": "none",
   "assets": [{
      "name": "jquery-js",
      "version": "1.11.0",
      "type": "js",
      "locations": {
         "cdn": "//cdnjs.cloudflare.com/ajax/libs/jquery/1.11.0/jquery.js",
         "webapp" : "/assets/js/jquery.js"
      }
   }]
}

where:

  • bundle is the name given to the bundle
  • parent is the name of the parent bundle.
  • assets is an array of asset, where:
    • name is the name given to the asset
    • version is the version of the asset
    • type is the asset type (among js and css)
    • locations is an array of locations, where a each location contains:
      • a location key, which is mapped to a location wrapper. Thanks to this key, Dandelion knows the fetching strategy to use for the asset.
      • a path, potentially processed depending on the corresponding location key.

By default, Dandelion uses a standard asset loader that retrieves all bundles from the /dandelion folder in your classpath.

For example, assuming you are in a Maven project, all bundles must locate under the src/main/resources/dandelion folder. 

project-root
|__ src
   |__ main
      |__ resources
         |__ dandelion
            |__ bundle1.json
            |__ bundle2.json
            |__ ...

Note that asset loaders are fully documented here.

Bundle hierarchy

You may have noticed the parent property of a bundle. Indeed, bundles can be assembled and form a bundle hierarchy. See the specific section for a complete overview of the bundle hierarchy.
