Introducing asset scopes

Asset and asset scopes are the main unit of work in Dandelion.

What is an asset?

An asset, or web resource, is basically a JavaScript or a CSS file, with some additional information.

By default, Dandelion uses JSON as a main representation format. Here is how an asset is defined:

  • name: name of the asset.
  • version: version of the asset.
  • type: type of the asset, among js or css.
  • locations: set of locations available for the asset.

For example, if we wanted to translate the v1.11.0 of jQuery that points to cdnjs, it would look like: 

{
    "name": "jquery-js",
    "version": "1.11.0",
    "type": "js",
    "locations": {
        "cdn": "//cdnjs.cloudflare.com/ajax/libs/jquery/1.11.0/jquery.js"
    }
}

multiple locations?

What is an asset scope?

An asset scope is just a set of assets, grouped together inside a .json file. The asset scope has 3 main propertiess:

  • scope: the scope name.
  • parent: the name of the parent scope
  • assets: the array of assets contained in this scope

Consider, for example, the jquery scope (coming from the jquery.json file).

{
    "scope" : "jquery",
    "parent": "none",
    "assets": [
      {
          "name": "jquery-js",
          "version": "1.11.0",
          "type": "js",
          "locations": {
              "cdn": "//cdnjs.cloudflare.com/ajax/libs/jquery/1.11.0/jquery.js"
          }
      }
    ]
}

Note that:

  • the jquery scope has a parent called none . It basically means that the jquery scope has no parent.
  • the jquery scope contains only one asset: the sources of jQuery v1.11.0.

Let's take another example: the DataTables jQuery plugin. The datatables scope would look like: 

{
    "scope" : "datatables",
    "parent": "jquery",
    "assets": [
        {
            "name": "datatables-js",
            "version": "1.9.4",
            "type": "js",
            "locations": {
                "cdn": "//ajax.aspnetcdn.com/ajax/jquery.dataTables/1.9.4/jquery.dataTables.js"
            }
        },
        {
            "name": "datatables-css",
            "version": "1.9.4",
            "type": "css",
            "locations": {
                "cdn": "//ajax.aspnetcdn.com/ajax/jquery.dataTables/1.9.4/css/jquery.dataTables.css"
            }
        }
    ]
}

This scope is essentially the same as the previous one with the exceptions that:

  • It has a parent scope: jquery. Pretty simple to understand, indeed the DataTables jQuery plugin needs jQuery to work.
  • This scope contains two assets: one JavaScript file and one CSS file, all pointing to a CDN.

Conclusion

As a conclusion, an asset scopes, or a scope to simplify the language is stored in a JSON file and contains some information:

  • a name
  • the name of its parent
  • a set of assets (JS and CSS files) that can come from different locations
