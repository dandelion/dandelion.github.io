Using configuration

You can also use the configuration properties to interact with the asset stack.

Note that all following properties have a global impact on the asset stack, i.e. in all your application.

Activate bundles permanently

Among all available properties, one can be use to activate a bundle permanently. It can be handy for example when you need to load a CSS file everywhere in your application.

For this purpose, just define the bundles.include in your dandelion.properties file.

For example, assuming your dandelion.properties file contains the following configuration: 

bundles.include=bootstrap2
The bootstrap2 bundle will be loaded into the asset stack permanently. As a consequence, all assets defined in this bundle will be loaded in all pages.
