Configuration loading

Preamble

Dandelion-Datatables internally uses a configuration file called dandelion-default.properties that contains configuration properties used by default.

These configuration properties can be overloaded in multiple ways. This section decribes how.

Loading strategy

You can overload the default configuration using a file named dandelion.properties. Dandelion will load it thanks to the ResourceBundle mechanism and using the following strategy:

  • First, check if the dandelion.configuration system property is defined.

    If it exists, Dandelion attempts to convert the value to a URL and use it to load a resource bundle called dandelion.properties

  • If no system property is defined, Dandelion tries to load a resource bundle still called dandelion.properties in classpath_root/dandelion/

  • If no dandelion.properties file is found, only the default configuration will be used.

Example using classpath configuration

Let's say you use Maven as a build tool. When reading configuration from classpath, you should strictly follow the convention, i.e. place the properties files under the classpath_root/dandelion/ folder. 

   MyApp
   |__ src
      |__ main
         |__ java
         |__ resources
            |__ logback.xml
            |__ ...
            |__ dandelion
               |__ dandelion.properties

Example using externalized configuration

Let's say you have the following files: 

   C:
   |__ Documents
      |__ MyApp
         |__ dandelion.properties
You just need to add -Ddandelion.configuration="C:\Documents\MyApp\" to the starting script of the server.

Note that when reading from externalized file, you don't have to place the properties files under the dandelion folder.
